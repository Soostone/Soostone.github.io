                                      Napkin fundamentals tutorial

                                      This is the first of a series of tutorials to get up and running with Napkin development. We do this through the example of building a simple Napkin data pipeline. The goal is that by the end of this tutorial, you’ll have a good idea of:

                                      • How to start a Napkin project
                                      • How to modify and verify and execute your Napkin project
                                      • How Napkin helps you build a real-life application fast.

                                      With that, let’s get started!

                                      Getting started

                                      • System requirements
                                      • Napkin installation

                                      • Working with Napkin

                                      After reviewing this topic, you can get started with Fundamental tutorial

                                      System requirements

                                      • Linux or macOS
                                      • A terminal application for command line interaction.
                                      • SQLite

                                      Napkin Installation

                                      To get started, follow steps outlined in Napkin Installation

                                      Install Chinook SQLite database

                                      For this tutorial uses SQLite backend and Chinook database as our test data.

                                      To install a local copy of SQLite Chinook database:

                                      #!/usr/bin/env bash
curl -L -o chinook-sql.db  https://github.com/lerocha/chinook-database/raw/master/ChinookDatabase/DataSources/Chinook_Sqlite.sqlite
echo "SELECT * FROM Artist" | sqlite3 chinook-sql.db ## verify chinook is installed

                                      Bootstrapping a Napkin project

                                      With both Napkin and Chinook installed, we are ready to create our first Napkin data pipeline project. The easiest way to bootstrap a Napkin project is utilizing the Napkin init CLI command. to create a new project based on Napkin’s default template:

                                      mkdir ./napkinFundamentalTutorial
cd ./napkinFundamentalTutorial
napkin init --project-name chinook-analytic

                                      The init command uses the Napkin embedded :basic template to create the following project directory structure:

                                      chinook-analytic
  ├── hie.yaml
  ├── README.md
  ├── specs
  │   └── spec.yaml
  └── sql
      └── example.sql

                                      Note: “” Napkin’s CLI is the point of entry for all typical workflows of data engineering and pipeline curation. napkin --help provides a comprehensive list of available Napkin commands.

                                      To see available options for the init command:

                                      napkin init --help

                                      Adding computations to the pipeline

                                      In this section of the tutorial, we add the following processing steps:

                                      • create ranking of Chinook sales to countries
                                      • use the raking to find:
                                        • top 10 countries with the most invoices
                                        • top 20 countries with the most invoices
                                      • create ranking of Chinook best cusomters
                                      • use the raking to find:
                                        • top 10 customers
                                        • top 20 customers

                                      To accomplish the above tasks we first modify the specs/spec.yaml to

                                      1- create managed tables for the rankings 2- manage the dependencies between the tables

                                      Then we use Napkin CLI to

                                      1- validate our project, napkin validate --spec-file specs/spec.yaml 2- show the dependencies, napkin run --dry-run --spec-file specs/spec.yaml 3 execute out pipeline, napkin run --spec-file specs/spec.yaml

                                      Napkin follows a common paradigm where source tables are not mutated by Napkin. Instead, Napkin creates a series of dependent tables, managed tables based on the SQL file in sql/ folder. Internally, Napkin uses specs/spec.yaml to create a DAG where the edges are references to fields from managed tables. The DAG is used to drive the Napkin execution plan.

                                      Create the rankings

                                      We’ll start by creating the SQL file:

                                      • sql/client_countries.sql
                                      -- client countries and total invoices
SELECT
    BillingCountry AS billingCountry,
    COUNT(*) AS Invoices
FROM
    Invoice
GROUP BY
    BillingCountry
                                      • sql/customers_total_purchase.sql
                                      SELECT
    C.Customerid,
    C.email,
    C.FirstName,
    C.LastName,
    SUM(I.TOTAL) AS Total_purchase
FROM
    Customer C
    JOIN Invoice I ON C.Customerid = I.Customerid
GROUP BY
    C.CustomerId
                                      • sql/top_performers.sql
                                      SELECT
    *
FROM
    
ORDER BY
     FETCH FIRST  ROWS ONLY

                                      We use Napkin’s variable interpolation for the top_performers generalize the top ranking computation

                                      Working with Napkin’s spec

                                      In this section, we will

                                      • modify spec.yaml
                                      • validate spec.yaml
                                      • show the managed tables dependencies spec.yaml
                                      • execute the napkin pipeline
                                      specs/spec.yaml

                                      Napkin’s Spec is a high level DSL in YAML configuration language. In this section we’ll author the project spec.yaml such that we can take advantage of napkin’s

                                      • Assertions, post-hook and pre-hook
                                      • interpolation
                                      • dependencies discovery

                                      With that lets modify the projects specs/spec.yaml to define and execute our pipeline:

                                      # yaml-language-server: $schema=http://soostone-napkin-public.s3-website.us-east-1.amazonaws.com/schema/schema.json

# Connect to Sqlite database:
backend: Sqlite
db_url: sqlite:.chinook-sql.db

tables:

## countries have the most Invoices
  client_countries:
    create_action:
      sql_file:
        source: client_countries.sql
    post_hooks:
      - assert_unique:
          table: client_countries
          columns: 
          - billingCountry

## top 10 countries have the most Invoices
  top10_client_countries:
    create_action:
      sql_file:
        source: top_performers.sql
        vars:
          table: client_countries
          max_limit: 10
          top_by: Invoices
          
  ## best customers
  cutomers_total_purchase:
    update_strategy: # TODO: let's add this to dependent tables
      - type: if_missing
      - type: with_dependency
    create_action:
      sql_file:
        source: customers_total_purchase.sql

  ## top 10 customers
  top10_customers:
    create_action:
      sql_file:
        source: top_performers.sql
        vars:
          table: cutomers_total_purchase
          max_limit: 10
          top_by: TOTAL_PURCHASE

                                      Next, we’ll validate the spec:

                                      napkin validate --spec-file ./specs/spec.yaml

                                      Then we use the Napkin dump command to visually verify the managed-tables dependencies:

                                      napkin dump -o ~/tmp/myProject --spec-file ./specs/spec.yaml

                                      Then we do dry-run to get an intution to what napkin is about to do:

                                      napkin run --dry-run  --spec-file ./specs/spec.yaml
